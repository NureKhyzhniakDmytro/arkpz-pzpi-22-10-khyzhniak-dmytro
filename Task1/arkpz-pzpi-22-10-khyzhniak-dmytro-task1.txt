Назва проекту: «Програмна система для управління вентиляцією».
1.1.	Background
Сучасні підприємства та житлові будівлі все частіше використовують автоматизовані системи для контролю та управління середовищем, щоб забезпечити комфортні умови для проживання та роботи. Одним із важливих аспектів є якість повітря в приміщеннях, яка безпосередньо впливає на здоров'я, продуктивність та комфорт користувачів. Погана вентиляція може призводити до накопичення вуглекислого газ, високої вологості, що створює сприятливі умови для розвитку грибків та бактерій, а також зниження якості повітря загалом.
Основна ідея проекту полягає в розробці програмної системи управління вентиляцією, яка за допомогою IoT-пристроїв автоматизує процес моніторингу та регулювання параметрів повітря в приміщеннях. Система призначена для збору даних про якість повітря, аналізу цих даних та автоматичного регулювання роботи вентиляційних пристроїв для підтримання заданих користувачем параметрів.
Метою розробки продукту є створення рішення, яке забезпечить:
•	Підтримання оптимальних умов у приміщеннях (температура, вологість, рівень CO2) для забезпечення комфорту та здоров'я людей.
•	Автоматизацію процесу контролю та управління вентиляцією, що знижує потребу у ручному втручанні користувачів.
•	Енергоефективність через раціональне використання вентиляційного обладнання.
•	Гнучкість у налаштуванні системи для різних типів приміщень та потреб користувачів (житлові приміщення, офіси, виробничі приміщення).
1.2.	Business Opportunity
Аналіз аналогів
Існують різні рішення для автоматизованого управління вентиляцією, зокрема:
•	Розумні системи дому (Google Nest, Ecobee), які моніторять базові параметри повітря.
•	Індустріальні системи (Honeywell BMS) для великих підприємств, які є дорогими у впровадженні.
•	Локальні системи контролю повітря, які надають тільки моніторинг без автоматизації.
Переваги нашого продукту
•	Гнучкість налаштувань для користувачів різних типів приміщень.
•	Автоматичне управління параметрами повітря, що зменшує ручне втручання.
•	Доступність і простота інтеграції для невеликих компаній та індивідуальних користувачів.
Модель монетизації
•	Продаж пристроїв для збору даних та управління вентиляцією.
•	Підписка на хмарний сервіс з доступом до розширених функцій.
•	Налаштування та підтримка для компаній, які потребують адаптації.
Цільовий ринок і перспективи
Основний ринок — малі та середні підприємства, а також власники будинків, які шукають доступне рішення для поліпшення якості повітря. Через зростаючий попит на автоматизацію IoT, продукт має перспективи для швидкого розширення та отримання частки ринку.
2.1	 Vision Statement
Програмна система управління вентиляцією призначена для автоматизації та дистанційного контролю за якістю повітря в приміщеннях. Вона забезпечує збір даних про параметри повітря (температура, вологість, рівень CO₂) за допомогою IoT-пристроїв, їхню обробку на сервері та автоматичне регулювання роботи вентиляційних пристроїв (Smart Devices) для підтримання оптимальних умов.
Система дозволяє компаніям та індивідуальним користувачам налаштовувати параметри вентиляції відповідно до своїх потреб, знижувати енергоспоживання, а також забезпечувати комфортні та безпечні умови для роботи та проживання. Користувачі можуть дистанційно керувати системою через веб-інтерфейс або мобільний додаток, отримувати сповіщення про критичні зміни параметрів та переглядати історію даних.
Головна мета продукту — забезпечити гнучке та зручне управління мікрокліматом у приміщеннях різного типу (офіси, виробничі приміщення, житлові будинки) за допомогою сучасних IoT-рішень, що сприяє економії енергії, підвищенню комфорту та безпеки.
2.2	 Major Features
MF-1: Автоматичне регулювання: система автоматично аналізує дані від IoT-пристроїв (сенсорів) і коригує роботу вентиляційних пристроїв для підтримки оптимальних умов у приміщенні.
MF-2: Дистанційне управління та моніторинг: користувачі можуть налаштовувати параметри роботи пристроїв та моніторити стан повітря в приміщеннях через веб-клієнт або мобільний додаток.
MF-3: Сповіщення про критичні зміни: система відправляє сповіщення користувачам у разі, якщо параметри повітря виходять за встановлені межі, що дозволяє швидко реагувати на проблеми.
MF-4: Індивідуальні налаштування порогових значень: користувачі можуть задавати порогові значення для параметрів повітря (температура, вологість, рівень CO2) залежно від потреб приміщення.
MF-5: Історія даних та аналітика: система зберігає дані про параметри повітря та дозволяє переглядати історію показників для аналізу та оптимізації налаштувань.
MF-6: Рольове управління доступом: адміністратори можуть налаштовувати систему та додавати нові пристрої, а співробітники можуть лише переглядати показники та отримувати сповіщення про критичні зміни.
MF-7: Додавання нових пристроїв та приміщень: користувачі можуть створювати приміщення, додавати до них датчики та вентиляційні пристрої, зв’язуючи їх між собою, а також надавати доступ співробітникам для моніторингу.
3.1	 Scope of Initial Release
1. Функціональність серверної частини
•	Управління користувачами та ролями: система надає можливість адмініструвати облікові записи користувачів, зокрема створення та управління ролями (адміністратори та співробітники).
•	Обробка даних з IoT-пристроїв: сервер приймає та обробляє дані від сенсорів IoT (температура, вологість, рівень CO2), визначає необхідність коригування роботи вентиляційних пристроїв.
•	Взаємодія з IoT-пристроями: сервер відправляє команди на вентиляційні пристрої залежно від оброблених даних.
•	Інтерфейси для адміністрування: надання API для управління налаштуваннями приміщень, порогових значень, пристроїв та доступу користувачів.
•	Зберігання історії даних: зберігання історичних даних про показники повітря для подальшого аналізу та відображення.
2. Функціональність IoT та Smart Device програмного застосунку:
•	Збір та передача даних:  сенсори збирають інформацію про параметри повітря та відправляють її на сервер з визначеною періодичністю.
•	Виконання команд: вентиляційні пристрої отримують команди від сервера та змінюють параметри роботи для забезпечення оптимального мікроклімату.
•	Сумісність з протоколами зв'язку: підтримка стандартних протоколів зв'язку для забезпечення взаємодії з сервером.
3. Функціональність веб-клієнта:
•	Двомовний інтерфейс: реалізація інтерфейсу українською та англійською мовами, з урахуванням інтернаціоналізації та локалізації.
•	Налаштування приміщень та пристроїв: адміністратори можуть створювати приміщення, додавати до них сенсори та вентиляційні пристрої, визначати порогові значення.
•	Моніторинг стану: відображення поточних показників параметрів повітря в реальному часі та надання доступу до історичних даних.
•	Керування доступом співробітників: надання доступу до моніторингу співробітникам на рівні приміщення для перегляду поточних показників і отримання сповіщень.
4. Функціональність мобільного клієнта:
•	Моніторинг та налаштування на мобільних пристроях: адміністратори можуть керувати системою з мобільного додатку, а співробітники отримувати сповіщення та переглядати поточні показники.
•	Підтримка сповіщень: відправлення push-сповіщень про критичні зміни параметрів повітря для швидкого реагування.
5. Інтернаціоналізація та локалізація:
•	Підтримка локалізації системи, включаючи кодування символів, формат дати та часу, різні одиниці вимірювання.
•	Підтримка англійської та української мов в інтерфейсі для зручності користувачів з різних регіонів.
6. Захист персональних даних:
•	Безпека передачі даних: використання протоколів шифрування для захисту даних, що передаються між сервером і IoT-пристроями, а також клієнтами.
•	Відповідність стандартам захисту даних: реалізація заходів для дотримання сучасних стандартів щодо захисту персональних даних користувачів.
7. Засоби адміністрування:
•	Управління даними та резервне копіювання: функціональність для створення резервних копій налаштувань і даних, можливість імпорту та експорту даних.
•	Моніторинг системи: надання адміністраторам можливості моніторингу роботи системи та перевірки стану підключених IoT-пристроїв.
3.2	Scope of Subsequent Releases
SR-1: Додавання підтримки нових типів сенсорів та пристроїв: розширення підтримки різних типів сенсорів (наприклад, сенсори пилу, летючих органічних сполук) та вентиляційних пристроїв, що дозволить підключати нові IoT-пристрої до системи.
SR-2: Розширення можливостей звітності та аналітики: впровадження більш гнучких звітів та інструментів аналітики для вивчення тенденцій якості повітря, що дозволить адміністраторам краще розуміти динаміку та налаштовувати систему відповідно до потреб.
SR-3: Розширення можливостей управління ролями: додавання нових ролей з більш детальним налаштуванням прав доступу, що дозволить більш гнучко розподіляти повноваження між співробітниками.
SR-4: Мобільний додаток для налаштування IoT-пристроїв: розробка мобільного додатка з можливістю налаштування та калібрування IoT-пристроїв безпосередньо на місці їх встановлення, що спростить процес запуску системи.
SR-5: Підтримка додаткових мов інтерфейсу: додавання підтримки нових мов для інтерфейсу, що розширить доступність системи для користувачів з різних регіонів.
SR-6: Розширення системи сповіщень: додавання можливості налаштовувати різні типи сповіщень (електронна пошта, SMS) для користувачів залежно від критичності подій та параметрів.
4.3	 Operating Environment
Система розгортається на Linux з використанням ASP.NET для створення веб-сервісів, PostgreSQL для управління даними та написана на C#. Використовуються Firebase Auth для автентифікації користувачів і Firebase Cloud Messaging для відправлення повідомлень. Вебсайт розробляється на Vite.js з TypeScript та застосуванням фреймворку Tailwind для стилізації.
IoT-компоненти базуються на Raspberry Pi з зовнішніми модулями для підключення сенсорів, що збирають дані про повітря. Управління вентиляцією реалізовано на окремому пристрої. 
Мобільний застосунок створений з використанням нативного Android SDK на Java для забезпечення управління системою та отримання сповіщень.
Вимоги до системи включають високу надійність, продуктивність, доступність та підтримку цілісності даних для безперебійного функціонування в багатозональних середовищах.
